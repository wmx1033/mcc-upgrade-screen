# STM32H743 串口屏烧录桥接工具（固定 115200）

> 本项目用于串口屏 whmi-wri 烧录：PC <-> USART1 <-> MCU <-> USART6 <-> 屏。
> 本场景所有波特率固定 115200，因此无需动态切换波特率；核心是保证透明转发与 0x05 ACK 节奏不被破坏。

## 成功标准（必须满足）
- **字节透明**：不插入/不丢失/不篡改/不重排任意字节（含 `0xFF 0xFF 0xFF` 与二进制数据）
- **ACK 透明**：屏返回 `0x05` 为单字节，必须原样回传给 PC（无结束符）
- **长包稳定**：连续发送 >= 1MB 数据不溢出、不死锁、不丢包
- **无污染输出**：禁止 printf/日志输出到 USART1/USART6

## 核心原则
- CubeMX 优先；Core 只改 USER CODE BEGIN/END；禁止改生成代码
- 数据阶段不得做任何解析/转义/打印

## 协议要点（仅用于理解节奏）
- `whmi-wri filesize,115200,res0` + `0xFF 0xFF 0xFF`
- 屏约 250ms 后回 `0x05`（单字节）
- 数据按 4096 字节/包发送，每包后屏回 `0x05`

## 推荐实现
- USART1/USART6：115200 8N1，中断接收 + RingBuffer（两方向独立，建议 >= 8KB/方向）
- TX：中断或 DMA 均可，保证吞吐，不要长临界区
- 可选“烧录模式”状态机：
  - 识别 `whmi-wri` 后开始统计 PC->屏转发字节数（filesize）与 4096 包边界
  - 在包边界处增加 ACK 超时保护（但不得向串口输出日志）
